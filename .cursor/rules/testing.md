# Стратегія тестування

## Інструменти
- pytest як фреймворк.
- pytest-cov та coverage.py для вимірювання покриття.
- unittest.mock (MagicMock) для ізоляції сервісів від сховищ.

## Що покриваємо
- Моделі: зміна залишку, резервування, переходи станів документів, валідація вхідних значень.
- Сервіси: основні сценарії та граничні випадки (недостатньо товару, неправильний стан, дубль артикулу).
- Шаблони: кожна стратегія окремо, спостерігачі на отримання подій, фабрика на коректний тип руху.
- Інтеграційні сценарії: повний цикл від поставки до списання через Application.

## Звіти
Налаштовано в pytest.ini. Після запуску генеруються:
- reports/junit.xml для інтеграції з SonarQube.
- reports/coverage.xml у форматі Cobertura для SonarQube.
- reports/htmlcov для візуального перегляду непокритих рядків.

Команда запуску:

    python -m pytest

## Цілі покриття
Мінімум 70 відсотків за вимогами Quality Gate. Орієнтовна кількість тестів близько 200 і більше. Для нового коду спершу пишемо тест, потім реалізацію.

## Поради для генерації тестів
- Назви тестів англійською, коментарі українською.
- Один тест перевіряє одну поведінку.
- Для перевірки взаємодії сервісу зі сховищем підставляй MagicMock замість реального репозиторію.
